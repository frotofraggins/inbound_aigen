## üìù Documentation Standards

### When to Create New Docs
**DO create new docs for:**
- Major new features (e.g., Phase 5 if significantly different)
- New subsystems (e.g., SEC data integration)
- Specialized operations guides

**DON'T create new docs for:**
- Minor updates (update existing docs)
- Status snapshots (use git commits)
- Debugging sessions (add to TROUBLESHOOTING_GUIDE)
- Session notes (use git commits or archive)

### How to Update Docs
1. **Update existing doc** if content belongs there
2. **Mark sections obsolete** rather than deleting (with "DEPRECATED" tags)
3. **Update "Last Updated" dates** at top of doc
4. **Keep DOCUMENTATION_INDEX current** when adding/removing docs

### Doc Naming Convention
- **UPPERCASE_SNAKE_CASE.md** for essential docs
- **lowercase-kebab-case.json** for config files
- **Descriptive names** that indicate content
